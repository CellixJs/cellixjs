# @cellix/graphql-codegen

Cellix graphql-codegen plugins and schema builder. This package provides two things:

1. **Build-time**: graphql-codegen plugins that generate static TypeScript files, replacing runtime glob-based file discovery and enabling rolldown to bundle the API without dynamic imports at startup.
2. **Runtime**: `buildCellixSchema` — a function that consumes the generated artifacts to construct the complete GraphQL schema on startup.

## Installation

```bash
pnpm add @cellix/graphql-codegen
```

For a reusable starting point, copy [`codegen.template.yml`](./codegen.template.yml) into your repo root as `codegen.yml` and then replace the example app/package names, schema paths, scalar mappings, and domain mappers for your project.

---

## `buildCellixSchema`

The main runtime export. Constructs an executable GraphQL schema by merging the Cellix base types (from `@cellix/graphql-core`) with application-specific type definitions and resolvers.

```typescript
import { buildCellixSchema } from '@cellix/graphql-codegen';
import { ocomGraphqlTypeDefs } from './schema-type-defs.generated.ts';
import { ocomGraphqlResolvers } from './resolver-manifest.generated.ts';

export const schema = buildCellixSchema(
  [...ocomGraphqlTypeDefs],
  [ocomGraphqlResolvers],
);
```

**Signature:**

```typescript
function buildCellixSchema<TContext = Record<string, unknown>>(
  additionalTypeDefs?: (string | DocumentNode) | Array<string | DocumentNode>,
  additionalResolvers?: IResolvers<unknown, TContext>[],
): GraphQLSchema
```

The function automatically includes:
- All GraphQL scalars from `graphql-scalars`
- The Cellix base schema (Query, Mutation, MongoBase, shared interfaces) from `@cellix/graphql-core`

## Plugins

### `@cellix/graphql-codegen/plugins/static-type-defs`

Reads all `.graphql` SDL files from a configurable directory and outputs a TypeScript constant with the raw SDL strings as a `readonly` array. Replaces `loadFilesSync` glob calls in schema builders.

**Config options:**

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `sourceDir` | `string` | ✅ | Path relative to repo root containing the `.graphql` files |
| `exportName` | `string` | ✅ | Name of the exported TypeScript constant |

**Output format:**

```typescript
// This file is auto-generated by graphql-codegen (static-type-defs-plugin).
// Do not edit directly.

export const myTypeDefs = [
  `type Query { ... }`, // path/to/file.graphql
] as const;
```

**Example (`codegen.yml`):**

```yaml
'./packages/my-app/graphql/src/schema/builder/schema-type-defs.generated.ts':
  plugins:
    - '@cellix/graphql-codegen/plugins/static-type-defs':
        sourceDir: 'packages/my-app/graphql/src/schema/types'
        exportName: 'myAppTypeDefs'
```

---

### `@cellix/graphql-codegen/plugins/resolver-manifest`

Scans a types directory for `.resolvers.ts` and `.permissions.ts` files and generates a TypeScript module with static imports for each. Replaces `loadResolversFromGlob` glob-based resolver discovery so rolldown can statically analyse all resolver imports at bundle time.

**Config options:**

| Option | Type | Required | Default | Description |
|--------|------|----------|---------|-------------|
| `typesDir` | `string` | ✅ | — | Path relative to repo root containing the resolver/permissions files |
| `resolversExportName` | `string` | ❌ | `'resolvers'` | Name of the exported resolvers array |
| `permissionsExportName` | `string` | ❌ | `'permissions'` | Name of the exported permissions array |

**Output format:**

```typescript
// This file is auto-generated by graphql-codegen (resolver-manifest-plugin).
// Do not edit directly.

import resolver0 from '../types/community.resolvers.ts';
import resolver1 from '../types/member.resolvers.ts';

export const resolvers = [
  resolver0,
  resolver1,
];

export const permissions = [];
```

**Example (`codegen.yml`):**

```yaml
'./packages/my-app/graphql/src/schema/builder/resolver-manifest.generated.ts':
  plugins:
    - '@cellix/graphql-codegen/plugins/resolver-manifest':
        typesDir: 'packages/my-app/graphql/src/schema/types'
        resolversExportName: 'myAppResolvers'
        permissionsExportName: 'myAppPermissions'
```

## Full `codegen.yml` example

Below is a complete `codegen.yml` showing how a Cellix project integrates both plugins alongside the standard graphql-codegen generates. Replace `my-app` and `myApp` with your application's namespace.

The same example is also available as [`codegen.template.yml`](./codegen.template.yml) in this package for copy/paste reuse across Cellix repos.

```yaml
overwrite: true

schema:
  - './packages/cellix/graphql-core/src/schema/**/**.graphql'
  - './packages/cellix/graphql-core/src/schema/core/graphql-tools-scalars.ts'
  - './packages/my-app/graphql/src/schema/**/**.graphql'

generates:
  # TypeScript resolver types
  './packages/my-app/graphql/src/schema/builder/generated.ts':
    config:
      contextType: '../context.ts#GraphContext'
      useIndexSignature: true
      useTypeImports: true
      enumsAsTypes: true
    plugins:
      - typescript
      - typescript-resolvers

  # Cellix core base type defs (static array for rolldown bundling)
  './packages/cellix/graphql-core/src/schema/base-type-defs.generated.ts':
    plugins:
      - '@cellix/graphql-codegen/plugins/static-type-defs':
          sourceDir: 'packages/cellix/graphql-core/src/schema'
          exportName: 'baseCellixTypeDefs'

  # Application SDL type defs (static array for rolldown bundling)
  './packages/my-app/graphql/src/schema/builder/schema-type-defs.generated.ts':
    plugins:
      - '@cellix/graphql-codegen/plugins/static-type-defs':
          sourceDir: 'packages/my-app/graphql/src/schema/types'
          exportName: 'myAppTypeDefs'

  # Resolver manifest (static imports replacing runtime glob discovery)
  './packages/my-app/graphql/src/schema/builder/resolver-manifest.generated.ts':
    plugins:
      - '@cellix/graphql-codegen/plugins/resolver-manifest':
          typesDir: 'packages/my-app/graphql/src/schema/types'
          resolversExportName: 'myAppResolvers'
          permissionsExportName: 'myAppPermissions'

hooks:
  afterAllFileWrite:
    - 'pnpm exec biome format --write'
```

## Generated files

Generated files follow the `*.generated.ts` naming convention and should be gitignored. Regenerate them at any time with:

```bash
pnpm gen
```

or in watch mode during development:

```bash
pnpm gen:watch
```
